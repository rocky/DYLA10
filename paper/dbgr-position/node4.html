Information to verify that source code matches object code

Dynamic languages often provide file-name information for source files, but are missing checksum-like information so that one can verify that what the programmer is looking at matches what the interpreter is running. This problem is especially serious if the front-end IDE is different from the one where the program is running (``remote debugging'') or when the programming language runs code from a bundled library.

This is easy to solve if the compiler runs a checksum as part of the scanning process, and then stores the result in the object.
